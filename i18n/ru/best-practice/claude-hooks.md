# Лучшие практики Hooks

![Last Updated](https://img.shields.io/badge/Last_Updated-Apr%2009%2C%202026-white?style=flat&labelColor=555) ![Version](https://img.shields.io/badge/Claude_Code-v2.1.96-blue?style=flat&labelColor=555)

Hooks в Claude Code — обработчики, управляемые событиями, которые выполняются вне агентного цикла на определённых событиях жизненного цикла.

## Обзор

Hooks — это пользовательские обработчики, которые выполняются **вне** агентного цикла — они не являются частью рассуждений Claude или цепочки вызовов инструментов. Они срабатывают на определённых событиях жизненного цикла и могут запускать скрипты, внедрять промпты, порождать агентов или вызывать HTTP-эндпоинты. Используйте hooks для добавления наблюдаемости, применения политик, автоматизации побочных эффектов и контроля решений без загрязнения контекста разговора.

## События Hook (27)

### Жизненный цикл сессии

| Событие | Срабатывает когда |
|---------|------------------|
| `SessionStart` | Начинается новая интерактивная сессия |
| `SessionEnd` | Сессия вот-вот завершится |
| `Setup` | Первый запуск Claude Code в проекте (или после `/init`) |

### Жизненный цикл инструментов

| Событие | Срабатывает когда |
|---------|------------------|
| `PreToolUse` | Перед выполнением вызова инструмента. Может заблокировать, разрешить или изменить вызов |
| `PostToolUse` | После успешного завершения вызова инструмента |
| `PostToolUseFailure` | После неудачного завершения вызова инструмента с ошибкой |
| `PermissionRequest` | Когда Claude запрашивает разрешение на инструмент, не одобренный пользователем заранее |
| `PermissionDenied` | Когда запрос на разрешение инструмента отклонён пользователем или политикой |

### Жизненный цикл агентов

| Событие | Срабатывает когда |
|---------|------------------|
| `SubagentStart` | Субагент (через инструмент `Agent(...)`) начинает выполнение |
| `SubagentStop` | Субагент завершает выполнение |
| `Stop` | **Основной** агент завершает свой ход ответа |
| `StopFailure` | Ход ответа основного агента завершается с ошибкой |

### Задачи / Команды

| Событие | Срабатывает когда |
|---------|------------------|
| `TeammateIdle` | Участник команды в сессии Agent Teams становится свободным и доступным |
| `TaskCreated` | Создаётся фоновая задача |
| `TaskCompleted` | Фоновая задача завершается |

### Контекст

| Событие | Срабатывает когда |
|---------|------------------|
| `PreCompact` | Перед выполнением сжатия контекста |
| `PostCompact` | После завершения сжатия контекста |
| `UserPromptSubmit` | Пользователь отправляет промпт (до того, как Claude его обработает) |
| `Notification` | Claude отправляет уведомление (напр., задача выполнена, требуется разрешение) |

### Конфигурация / Среда

| Событие | Срабатывает когда |
|---------|------------------|
| `ConfigChange` | Файл настроек изменён |
| `CwdChanged` | Рабочий каталог изменился (напр., через `/add-dir`) |
| `FileChanged` | Отслеживаемый файл изменён на диске |
| `InstructionsLoaded` | Файлы CLAUDE.md или rules загружены в контекст |

### Worktree

| Событие | Срабатывает когда |
|---------|------------------|
| `WorktreeCreate` | Создаётся git worktree для изолированного агента |
| `WorktreeRemove` | Git worktree очищается |

### MCP / Elicitation

| Событие | Срабатывает когда |
|---------|------------------|
| `Elicitation` | Claude нужен структурированный ввод от пользователя через MCP elicitation |
| `ElicitationResult` | Пользователь отвечает на MCP elicitation |

## Типы Hook (4)

| Тип | Синтаксис | Описание |
|-----|-----------|----------|
| `command` | `{"type": "command", "command": "python3 script.py"}` | Выполняет shell-команду. Получает данные события через переменные окружения. Stdout внедряется обратно в контекст (для `PreToolUse`/`Stop`). Ненулевой код выхода блокирует действие (для `PreToolUse`) |
| `prompt` | `{"type": "prompt", "prompt": "Проверь это на проблемы безопасности"}` | Внедряет промпт в контекст Claude в точке hook. Полезно для добавления инструкций или ограничений |
| `agent` | `{"type": "agent", "agent": "security-reviewer"}` | Порождает субагента для обработки события. Агент работает в собственном контексте и может использовать инструменты |
| `http` | `{"type": "http", "url": "https://example.com/webhook"}` | Отправляет данные события на HTTP-эндпоинт через POST. Тело ответа внедряется обратно в контекст |

## Иерархия конфигурации

Hooks можно определять на нескольких уровнях. Более высокие уровни переопределяют более низкие для одного и того же события:

| Уровень | Расположение | Область действия |
|---------|-------------|-----------------|
| **Управляемый** | `managed-settings.json` (MDM / Registry) | Установлен организацией, не может быть переопределён |
| **Локальные настройки** | `.claude/settings.local.json` | Личные переопределения проекта (игнорируются git) |
| **Общие настройки** | `.claude/settings.json` | Общие hooks проекта для команды |
| **Глобальные настройки** | `~/.claude/settings.json` | Личные значения по умолчанию для всех проектов |
| **Frontmatter агента** | `.claude/agents/<name>.md` поле `hooks:` | Ограничен конкретным субагентом |
| **Frontmatter skill** | `.claude/skills/<name>/SKILL.md` поле `hooks:` | Ограничен конкретным skill |

Чтобы отключить все hooks, установите `"disableAllHooks": true` в `.claude/settings.local.json`.

## Синтаксис Matcher

Matchers фильтруют, какие вызовы инструментов запускают hook. Они применяются к событиям `PreToolUse`, `PostToolUse` и `PostToolUseFailure`.

| Matcher | Соответствие |
|---------|-------------|
| `"Bash"` | Все вызовы инструмента Bash |
| `"Read"` | Все вызовы инструмента Read |
| `"Write"` | Все вызовы инструмента Write |
| `"Edit"` | Все вызовы инструмента Edit |
| `"Glob"` | Все вызовы инструмента Glob |
| `"Grep"` | Все вызовы инструмента Grep |
| `"Agent"` | Все вызовы инструмента Agent (субагент) |
| `"Skill"` | Все вызовы инструмента Skill |
| `"WebFetch"` | Все вызовы инструмента WebFetch |
| `"WebSearch"` | Все вызовы инструмента WebSearch |
| `"mcp__<server>__<tool>"` | Конкретный инструмент MCP на конкретном сервере |
| `"*"` | Все вызовы инструментов (wildcard) |

Несколько matchers можно объединить в массив: `["Bash", "Write", "Edit"]`.

## Переменные окружения

Hooks типа command получают данные события через переменные окружения:

| Переменная | Доступна в | Описание |
|-----------|-----------|----------|
| `CLAUDE_HOOK_EVENT` | Все | Имя события (напр., `PreToolUse`, `Stop`) |
| `CLAUDE_HOOK_TOOL_NAME` | События инструментов | Вызываемый инструмент (напр., `Bash`, `Write`) |
| `CLAUDE_HOOK_TOOL_INPUT` | `PreToolUse` | JSON-строка входных параметров инструмента |
| `CLAUDE_HOOK_TOOL_OUTPUT` | `PostToolUse` | JSON-строка вывода инструмента |
| `CLAUDE_HOOK_ERROR` | События ошибок | Сообщение об ошибке неудавшейся операции |
| `CLAUDE_HOOK_SESSION_ID` | Все | Идентификатор текущей сессии |
| `CLAUDE_HOOK_CWD` | Все | Текущий рабочий каталог |
| `CLAUDE_HOOK_PROJECT_DIR` | Все | Корневой каталог проекта |
| `CLAUDE_HOOK_AGENT_NAME` | События агентов | Имя задействованного субагента |
| `CLAUDE_HOOK_MATCHED` | События инструментов | Matcher, который запустил этот hook |

## Практические примеры

### Звуковое уведомление при завершении задачи

```json
{
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "afplay ~/.claude/hooks/sounds/done.mp3",
        "async": true
      }
    ]
  }
}
```

### Логирование всех вызовов инструментов

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "*",
        "type": "command",
        "command": "echo \"$(date -u +%Y-%m-%dT%H:%M:%SZ) $CLAUDE_HOOK_TOOL_NAME\" >> ~/.claude/logs/tools.log",
        "async": true
      }
    ]
  }
}
```

### Аудит безопасности — блокировка опасных команд

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "type": "command",
        "command": "python3 .claude/hooks/scripts/security-check.py"
      }
    ]
  }
}
```

Скрипт проверяет `CLAUDE_HOOK_TOOL_INPUT` на опасные паттерны (`rm -rf /`, `curl | bash`, `chmod 777`). Ненулевой код выхода блокирует вызов инструмента.

### Автоформатирование при записи файлов

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": ["Write", "Edit"],
        "type": "command",
        "command": "prettier --write \"$CLAUDE_HOOK_TOOL_INPUT\" 2>/dev/null || true",
        "async": true
      }
    ]
  }
}
```

## Паттерны управления решениями

Hooks `PreToolUse` могут возвращать сигналы решений через stdout или код выхода:

| Паттерн | Механизм | Эффект |
|---------|----------|--------|
| **Allow** | Код выхода `0`, stdout содержит `ALLOW` | Вызов инструмента продолжается без запроса разрешения |
| **Deny** | Ненулевой код выхода | Вызов инструмента блокируется, показывается сообщение об ошибке |
| **Ask** | Код выхода `0`, stdout содержит `ASK` | Пользователю показывается стандартный запрос разрешения |
| **Defer** | Код выхода `0`, нет вывода или пустой stdout | Передаётся обработке разрешений по умолчанию |

Для hooks типа `prompt` и `agent` внедрённый ответ направляет поведение Claude, а не контролирует выполнение напрямую.

## Лучшие практики

1. **Используйте `async: true` для побочных эффектов** — Звуковые уведомления, логирование и телеметрия никогда не должны блокировать агентный цикл. Устанавливайте `"async": true` на hooks, которым не нужно влиять на поведение Claude.

2. **Используйте `once: true` для hooks сессии** — Hooks `SessionStart`, выполняющие настройку (установка зависимостей, проверка среды), должны запускаться только один раз за сессию.

3. **Сохраняйте тайм-ауты короткими** — Синхронные hooks блокируют ответ Claude. Если ваш hook занимает более 2-3 секунд, сделайте его асинхронным или оптимизируйте скрипт.

4. **Ограничивайте hooks с помощью matchers** — Не запускайте hooks на каждом вызове инструмента, когда вам нужны только определённые инструменты. Используйте matchers для снижения накладных расходов.

5. **Используйте `command` для контроля, `prompt` для направления** — Когда нужен жёсткий контроль allow/deny, используйте hook типа command с кодами выхода. Когда нужно повлиять на подход Claude, используйте hook типа prompt.

6. **Тестируйте hooks изолированно** — Запускайте скрипты hooks вручную с примерными переменными окружения перед добавлением в настройки.

7. **Логируйте ошибки hooks** — Оборачивайте команды hooks в логику обработки ошибок, записывающую в файл журнала, чтобы тихие сбои были обнаружимы.

## Типичные ошибки

| Ошибка | Подробности |
|--------|------------|
| **Путаница `Stop` и `SubagentStop`** | `Stop` срабатывает только для **основного** агента. Используйте `SubagentStop` для событий завершения субагента. В ранних версиях `Stop` назывался `AgentStop` — он был переименован |
| **Накладные расходы на порождение процессов** | Каждый hook типа `command` порождает новый shell-процесс. Для высокочастотных событий, таких как `PostToolUse` с wildcard matcher, это добавляет задержку. Рассмотрите использование `async: true` или объединение логики в один скрипт |
| **Забытый matcher** | Hook `PreToolUse` без поля `matcher` срабатывает для **всех** вызовов инструментов. Всегда устанавливайте matcher, если только вы намеренно не хотите универсального покрытия |
| **Синхронные hooks блокируют ответы** | Медленный синхронный hook на `Stop` задерживает отображение ответа Claude пользователю. Всегда делайте некритичные hooks асинхронными |
| **Вывод hook загрязняет контекст** | Stdout синхронных hooks типа command внедряется в контекст Claude. Сохраняйте вывод минимальным или перенаправляйте в файл журнала |
| **Сюрпризы иерархии настроек** | Hook, определённый в `.claude/settings.json`, может быть переопределён `.claude/settings.local.json`. Проверяйте оба файла при отладке hook, который не срабатывает |
