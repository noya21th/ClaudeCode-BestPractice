## 2. CLAUDE.md в больших монорепозиториях

При работе с Claude Code в монорепозитории важно понимать, как файлы CLAUDE.md загружаются в контекст, чтобы эффективно организовать инструкции проекта.

<p align="center">
  <a href="https://x.com/bcherny/status/2016339448863355206"><img src="../../../best-practice/assets/claude-memory/claude-memory-monorepo.jpg" alt="Загрузка CLAUDE.md в монорепозиториях" width="600"></a>
</p>

### Два механизма загрузки

Claude Code использует два различных механизма загрузки файлов CLAUDE.md:

#### Загрузка предков (ВВЕРХ по дереву директорий)

При запуске Claude Code проходит **вверх** от текущей рабочей директории к корню файловой системы и загружает каждый найденный CLAUDE.md. Эти файлы загружаются **сразу при запуске**.

#### Загрузка потомков (ВНИЗ по дереву директорий)

Файлы CLAUDE.md в поддиректориях ниже текущей рабочей директории **НЕ загружаются при запуске**. Они включаются только когда Claude читает файлы в этих поддиректориях во время сессии. Это называется **ленивой загрузкой**.

### Пример структуры монорепозитория

Рассмотрим типичный монорепозиторий с отдельными директориями для разных компонентов:

```
/mymonorepo/
├── CLAUDE.md          # Инструкции корневого уровня (общие для всех компонентов)
├── frontend/
│   └── CLAUDE.md      # Инструкции для фронтенда
├── backend/
│   └── CLAUDE.md      # Инструкции для бэкенда
└── api/
    └── CLAUDE.md      # Инструкции для API
```

### Сценарий 1: Запуск Claude Code из корневой директории

При запуске Claude Code из `/mymonorepo/`:

```bash
cd /mymonorepo
claude
```

| Файл | Загружен при запуске? | Причина |
|------|----------------------|---------|
| `/mymonorepo/CLAUDE.md` | Да | Это ваша текущая рабочая директория |
| `/mymonorepo/frontend/CLAUDE.md` | Нет | Загружается только при чтении/редактировании файлов в `frontend/` |
| `/mymonorepo/backend/CLAUDE.md` | Нет | Загружается только при чтении/редактировании файлов в `backend/` |
| `/mymonorepo/api/CLAUDE.md` | Нет | Загружается только при чтении/редактировании файлов в `api/` |

### Сценарий 2: Запуск Claude Code из директории компонента

При запуске Claude Code из `/mymonorepo/frontend/`:

```bash
cd /mymonorepo/frontend
claude
```

| Файл | Загружен при запуске? | Причина |
|------|----------------------|---------|
| `/mymonorepo/CLAUDE.md` | Да | Это директория-предок |
| `/mymonorepo/frontend/CLAUDE.md` | Да | Это ваша текущая рабочая директория |
| `/mymonorepo/backend/CLAUDE.md` | Нет | Другая ветвь дерева директорий |
| `/mymonorepo/api/CLAUDE.md` | Нет | Другая ветвь дерева директорий |

### Ключевые выводы

1. **Предки всегда загружаются при запуске** — Claude проходит ВВЕРХ по дереву директорий и загружает все найденные файлы CLAUDE.md. Это обеспечивает доступ к инструкциям корневого уровня для всего репозитория.

2. **Потомки загружаются лениво** — файлы CLAUDE.md в поддиректориях загружаются только при взаимодействии с файлами в этих поддиректориях. Это предотвращает засорение сессии нерелевантным контекстом.

3. **Соседи никогда не загружаются** — если вы работаете в `frontend/`, `backend/CLAUDE.md` или `api/CLAUDE.md` не будут загружены в контекст.

4. **Глобальный CLAUDE.md** — вы также можете разместить CLAUDE.md в `~/.claude/CLAUDE.md` в домашней папке, который применяется ко ВСЕМ сессиям Claude Code независимо от проекта.

### Почему этот дизайн работает для монорепозиториев

- **Общие инструкции распространяются вниз** — CLAUDE.md корневого уровня содержит соглашения для всего репозитория, стандарты кода и общие паттерны, применяемые везде.

- **Инструкции для компонентов остаются изолированными** — фронтенд-разработчикам не нужны бэкенд-специфичные инструкции, засоряющие контекст, и наоборот.

- **Контекст оптимизирован** — благодаря ленивой загрузке файлов CLAUDE.md потомков, Claude Code избегает загрузки потенциально сотен килобайтов нерелевантных инструкций при запуске.

### Лучшие практики

1. **Помещайте общие соглашения в корневой CLAUDE.md** — стандарты кода, формат сообщений коммитов, шаблоны PR и другие рекомендации для всего репозитория.

2. **Помещайте специфичные инструкции в CLAUDE.md компонента** — паттерны фреймворка, архитектура компонентов, соглашения по тестированию, уникальные для данного компонента.

3. **Используйте CLAUDE.local.md для персональных предпочтений** — добавьте в `.gitignore` для инструкций, которые не нужно делить с командой.
