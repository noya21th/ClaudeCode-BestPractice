## Быстрая диагностика

Выполните эти команды первым делом, когда что-то не работает:

| Команда | Что проверяет |
|---------|---------------|
| `claude --version` | Установленная версия — сравните с [последним релизом](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) |
| `/doctor` | Состояние установки, версия Node.js, аутентификация, валидность настроек |
| `/status` | Текущая модель, аккаунт, подключение к API, уровень подписки |
| `/usage` | Лимиты плана, статус ограничений запросов, оставшаяся квота |
| `/context` | Использование контекстного окна, загруженные файлы памяти, активные Skill |
| `/hooks` | Активные конфигурации Hook и их источники |
| `/mcp` | Статус подключения MCP-серверов |

## Проблемы с установкой

### Claude Code не устанавливается

**Симптомы**: `npm install -g @anthropic-ai/claude-code` завершается с ошибками прав доступа или конфликтами версий.

**Причина**: Слишком старая версия Node.js, проблемы с правами npm или конфликтующие глобальные пакеты.

**Решение**:

```bash
# Проверьте версию Node.js (требуется 18+)
node --version

# Если версия слишком старая, обновите через nvm
nvm install 20
nvm use 20

# Исправьте права npm (macOS/Linux)
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH  # Добавьте в ~/.zshrc или ~/.bashrc

# Чистая установка
npm install -g @anthropic-ai/claude-code
```

### Команда claude не найдена

**Симптомы**: Терминал выводит `claude: command not found` после успешной установки.

**Причина**: Директория глобальных бинарников npm не добавлена в PATH.

**Решение**:

```bash
# Найдите, куда npm устанавливает глобальные бинарники
npm config get prefix

# Добавьте в профиль шелла (~/.zshrc, ~/.bashrc)
export PATH="$(npm config get prefix)/bin:$PATH"

# Перезагрузите шелл
source ~/.zshrc
```

## Проблемы с аутентификацией

### API-ключ не распознаётся

**Симптомы**: Claude Code запускается, но не может выполнять вызовы API. Сообщения об ошибках упоминают аутентификацию или недействительный ключ.

**Причина**: Отсутствующий или истёкший API-ключ. Неправильная переменная окружения.

**Решение**:

```bash
# Повторная аутентификация
claude /login

# Или задайте API-ключ напрямую
export ANTHROPIC_API_KEY="sk-ant-..."

# Для пользователей Bedrock
export CLAUDE_CODE_USE_BEDROCK=1
claude /setup-bedrock
```

### Ошибки ограничения запросов

**Симптомы**: Ошибки `429 Too Many Requests`. Claude перестаёт отвечать посреди задачи.

**Причина**: Превышены лимиты плана. Слишком много одновременных сессий или Agent.

**Решение**:

```bash
# Проверьте текущее использование
/usage

# Варианты:
# 1. Подождите сброса лимита (проверьте заголовок Retry-After)
# 2. Переключитесь на более лёгкую модель для массовых задач
/model haiku

# 3. Уменьшите количество одновременных Agent
# 4. Включите дополнительное использование
/extra-usage
```

## Проблемы с производительностью

### Медленные ответы

**Симптомы**: Claude отвечает более 30 секунд. Длительные паузы между вызовами инструментов.

**Причина**: Слишком большое контекстное окно. Загружено много Skill. Синхронные Hook блокируют ответы.

**Решение**:

```bash
# Проверьте использование контекста
/context

# Сожмите, если выше 50%
/compact

# Проверьте медленные синхронные Hook
/hooks
# Сделайте некритичные Hook асинхронными: "async": true

# Уменьшите количество загруженных Skill — проверьте, не активируются ли ненужные автоматически
/skills
```

### Высокий расход токенов

**Симптомы**: Квота расходуется быстрее ожидаемого. `/usage` показывает высокое потребление.

**Причина**: Большие файлы CLAUDE.md. Много предзагруженных Skill. Порождение Agent с полным наследованием контекста. Отсутствие сжатия.

**Решение**:

1. Ограничьте CLAUDE.md до 200 строк
2. Используйте `user-invocable: false` для Skill, которые должны предзагружаться только определёнными Agent
3. Установите `model: haiku` для Agent, не требующих полных возможностей
4. Регулярно сжимайте контекст с помощью `/compact`
5. Используйте `effort: low` или `effort: medium` для рутинных задач

## Проблемы с Hook

### Hook не срабатывает

**Симптомы**: Вы настроили Hook в settings, но он никогда не выполняется.

**Причина**: Неправильный файл настроек. Отсутствующий или несовпадающий matcher. Hook отключён. Переопределение через иерархию настроек.

**Решение**:

```bash
# 1. Проверьте, какие Hook активны и их источники
/hooks

# 2. Убедитесь, что Hook находится в правильном файле настроек
# Командные Hook: .claude/settings.json
# Личные Hook: .claude/settings.local.json
# Глобальные Hook: ~/.claude/settings.json

# 3. Проверьте, не отключены ли Hook глобально
# Ищите "disableAllHooks": true в любом файле настроек

# 4. Убедитесь, что matcher точно совпадает с именем инструмента
# "Bash" а не "bash", "Write" а не "write"

# 5. Проверьте иерархию настроек — файл с более высоким приоритетом может переопределять
```

### Hook блокирует ответы

**Симптомы**: Длительные паузы после вызовов инструментов. Claude выглядит зависшим на несколько секунд.

**Причина**: Синхронный Hook выполняет медленную команду. Сетевой вызов в неасинхронном Hook.

**Решение**:

```json
// Добавьте "async": true для неблокирующих Hook
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "*",
        "type": "command",
        "command": "python3 log.py",
        "async": true
      }
    ]
  }
}
```

### Таймаут Hook

**Симптомы**: Hook запускается, но результат никогда не используется. Предупреждение о таймауте Hook в логах.

**Причина**: Команда Hook выполняется слишком долго. Внешний сервис недоступен. Скрипт зависает.

**Решение**: Протестируйте команду Hook вручную. Добавьте таймаут в сам скрипт. Убедитесь, что внешние сервисы доступны. Рассмотрите возможность сделать Hook асинхронным, если результат некритичен.

## Проблемы с MCP

### MCP-сервер не подключается

**Симптомы**: MCP-инструменты недоступны. `/mcp` показывает сервер как отключённый.

**Причина**: Бинарник сервера не найден. Неправильная конфигурация. Конфликт портов. Отсутствуют переменные окружения.

**Решение**:

```bash
# 1. Проверьте статус MCP
/mcp

# 2. Проверьте конфигурацию сервера в .mcp.json или settings.json
# Убедитесь, что путь к команде правильный и файл исполняемый

# 3. Проверьте, заданы ли необходимые переменные окружения
# MCP-серверы часто требуют API-ключи в .env

# 4. Попробуйте перезапустить MCP-подключение
# Закройте и перезапустите Claude Code

# 5. Протестируйте MCP-сервер вручную
npx @modelcontextprotocol/server-name
```

### Отказ в доступе MCP

**Симптомы**: MCP-инструменты отображаются, но при вызове выдают ошибки разрешений.

**Причина**: Инструмент отсутствует в списке разрешённых. Режим разрешений слишком строгий.

**Решение**: Добавьте MCP-инструмент в список `allowedTools` в настройках или одобрите его при запросе. Для Agent включите `mcp__<server>__<tool>` в поле `tools:`.

## Проблемы с Agent

### Agent не вызывается автоматически

**Симптомы**: Вы создали Agent с описанием, содержащим `PROACTIVELY`, но Claude никогда не порождает его автоматически.

**Причина**: Описание недостаточно чётко указывает, когда вызывать. Инструмент Agent недоступен в текущем контексте. Другой механизм обрабатывает задачу.

**Решение**:

1. Убедитесь, что поле `description` использует `PROACTIVELY` и чётко описывает условие срабатывания
2. Проверьте, что файл Agent находится в `.claude/agents/` с расширением `.md`
3. Убедитесь, что Claude имеет доступ к инструменту `Agent` (не ограничен полем `tools:` родительского Agent)

### Agent запускает Bash вместо инструмента Agent

**Симптомы**: Claude выполняет `claude --agent my-agent` через Bash вместо использования инструмента `Agent(...)`.

**Причина**: В определении Agent используется расплывчатая формулировка вроде "launch" или "run", которую Claude интерпретирует как shell-команду.

**Решение**: В вашем CLAUDE.md или инструкциях Agent явно укажите: "Use the Agent tool, not bash commands, to spawn subagents." Избегайте терминов вроде "launch" или "execute" в описаниях Agent.

### Agent превышает maxTurns

**Симптомы**: Agent останавливается посреди задачи с неполным результатом.

**Причина**: Значение `maxTurns` слишком мало для сложности задачи. Agent зацикливается или повторяет попытки.

**Решение**: Увеличьте `maxTurns` во frontmatter Agent. Проверьте поведение Agent на предмет ненужных циклов. Разбейте сложные задачи на более мелкие подзадачи.

## Проблемы со Skill

### Skill не отображается в /menu

**Симптомы**: Вы создали Skill, но он не появляется при вводе `/`.

**Причина**: Установлено `user-invocable: false`. Файл Skill находится не в том месте. Синтаксическая ошибка в YAML frontmatter.

**Решение**:

```bash
# 1. Проверьте список Skill
/skills

# 2. Проверьте расположение файла: .claude/skills/<name>/SKILL.md

# 3. Проверьте frontmatter — удалите user-invocable: false
# или установите user-invocable: true

# 4. Проверьте YAML — типичные ошибки:
# - Отсутствуют разделители ---
# - Неправильные отступы
# - Неэкранированные спецсимволы
```

### Skill не обнаруживается автоматически

**Симптомы**: Skill существует и имеет описание, но Claude никогда не вызывает его автоматически.

**Причина**: Установлено `disable-model-invocation: true`. Описание не совпадает с контекстом, с которым сталкивается Claude. Слишком много Skill конкурируют за один и тот же триггер.

**Решение**:

1. Удалите `disable-model-invocation: true`, если требуется автообнаружение
2. Напишите понятное `description` с ключевыми словами, которые Claude встретит естественным образом
3. Уменьшите количество Skill — слишком много автообнаруживаемых Skill создаёт неоднозначность

## Проблемы с памятью

### CLAUDE.md не загружается

**Симптомы**: Инструкции в CLAUDE.md игнорируются. Claude не следует соглашениям проекта.

**Причина**: Файл находится в дочерней директории (ленивая загрузка, а не нетерпеливая). Опечатка в имени файла. Переопределение через управляемую политику.

**Решение**:

```bash
# 1. Проверьте загруженные файлы памяти
/context

# 2. Убедитесь, что имя файла точно "CLAUDE.md" (с учётом регистра)

# 3. Разберитесь в правилах загрузки:
# Предки (вверх от cwd): загружаются нетерпеливо при старте
# Потомки (вниз от cwd): загружаются лениво при обращении к файлам в той директории
# Братья: никогда не загружаются

# 4. Если файл должен загружаться всегда, переместите его в директорию-предка
# или сошлитесь на него из корневого CLAUDE.md
```

### CLAUDE.local.md попадает в коммиты

**Симптомы**: Личные предпочтения появляются у коллег. `.claude/settings.local.json` в истории git.

**Причина**: Локальные файлы не добавлены в `.gitignore`.

**Решение**:

```bash
# Добавьте в .gitignore
echo "CLAUDE.local.md" >> .gitignore
echo ".claude/settings.local.json" >> .gitignore

# Удалите из отслеживания, если уже закоммичены
git rm --cached CLAUDE.local.md
git rm --cached .claude/settings.local.json
```

## Проблемы с Git

### Очистка рабочих деревьев

**Симптомы**: Устаревшие git worktree остаются после завершения Agent. Дисковое пространство растёт.

**Причина**: Agent с `isolation: worktree` упал или был прерван до очистки.

**Решение**:

```bash
# Список рабочих деревьев
git worktree list

# Удалите устаревшие рабочие деревья
git worktree prune

# Принудительно удалите конкретное рабочее дерево
git worktree remove /path/to/worktree --force
```

### Автор коммита

**Симптомы**: Git-коммиты приписываются неправильному автору, когда Claude Code их создаёт.

**Причина**: Автор Git не настроен для сессий Claude Code.

**Решение**:

```bash
# Установите автора git для проекта
git config user.name "Your Name"
git config user.email "your@email.com"

# Или используйте флаг --author в Hook/скриптах
git commit --author="Your Name <your@email.com>" -m "message"
```
