## Обзор типов памяти

Claude Code имеет четыре различных системы памяти, каждая из которых служит своей цели:

| Тип памяти | Расположение | Область действия | Постоянство | Когда загружается |
|------------|-------------|-----------------|-------------|-------------------|
| **CLAUDE.md** | Директории проекта | Для проекта, иерархическая | Постоянная (файловая) | При старте сессии (предки) или при обращении к файлам (потомки) |
| **Автопамять** | `~/.claude/memory/` и `~/.claude/projects/<hash>/memory/` | Глобальная для пользователя или для проекта | Постоянная (файловая) | При старте сессии |
| **Память Agent** | `~/.claude/agent-memory/<agent>/` или `.claude/agent-memory/<agent>/` | Для Agent, с областью по полю `memory:` | Постоянная (файловая) | При запуске Agent |
| **Правила** | `.claude/rules/*.md` и `~/.claude/rules/*.md` | Привязана к паттернам файлов | Постоянная (файловая) | При обращении к файлам, совпадающим с glob-паттернами |

## Иерархия CLAUDE.md

### Расположение файлов

| Уровень | Путь | Общий | Описание |
|---------|------|-------|----------|
| **Глобальный** | `~/.claude/CLAUDE.md` | Нет (личный) | Применяется ко всем сессиям Claude Code на этой машине |
| **Корень проекта** | `./CLAUDE.md` | Да (в git) | Командные общие инструкции для всего проекта |
| **Локальный проекта** | `./CLAUDE.local.md` | Нет (в .gitignore) | Личные переопределения для конкретного проекта |
| **Компонентный** | `./frontend/CLAUDE.md` | Да (в git) | Инструкции для конкретного компонента |
| **Вложенный** | `./src/api/v2/CLAUDE.md` | Да (в git) | Инструкции для глубоко вложенной директории |

### Механизмы загрузки

Claude Code использует два различных механизма для обнаружения файлов CLAUDE.md:

#### Загрузка предков (вверх по дереву) — нетерпеливая

При старте сессии Claude проходит **вверх** от текущей рабочей директории до корня файловой системы и загружает каждый найденный CLAUDE.md. Они загружаются **немедленно при старте**.

```
/home/user/projects/myapp/src/   ← cwd
/home/user/projects/myapp/       ← CLAUDE.md загружен (предок)
/home/user/projects/             ← CLAUDE.md загружен (предок)
/home/user/                      ← CLAUDE.md загружен, если существует (предок)
~/.claude/                       ← CLAUDE.md загружен (глобальный)
```

#### Загрузка потомков (вниз по дереву) — ленивая

Файлы CLAUDE.md в поддиректориях ниже cwd **не** загружаются при старте. Они подключаются только тогда, когда Claude читает или редактирует файлы в этих поддиректориях во время сессии.

```
/myapp/                 ← cwd, CLAUDE.md загружен при старте
/myapp/frontend/        ← CLAUDE.md загружен при обращении к файлам frontend/
/myapp/backend/         ← CLAUDE.md загружен при обращении к файлам backend/
/myapp/backend/api/     ← CLAUDE.md загружен при обращении к файлам backend/api/
```

#### Ключевые правила

- **Предки всегда загружаются при старте** — нетерпеливо, вверх от cwd
- **Потомки загружаются лениво** — только при обращении к файлам в этой директории
- **Братья никогда не загружаются** — работа в `frontend/` не загружает `backend/CLAUDE.md`
- **Глобальный загружается всегда** — `~/.claude/CLAUDE.md` применяется к каждой сессии

## Стратегии для монорепозиториев

### Корневой CLAUDE.md — общие соглашения

Корневой файл должен быть сфокусирован на командных стандартах, применимых повсеместно:

```markdown
# CLAUDE.md (корневой)

## Стандарты кода
- Строгий режим TypeScript везде
- Никаких типов `any` без комментариев с обоснованием
- Предпочтительны функциональные компоненты с хуками

## Соглашения по Git
- Конвенциональные коммиты: feat/fix/chore/docs/refactor
- Squash merge в main

## Тестирование
- Каждый PR должен включать тесты
- Минимум 80% покрытия для нового кода
```

### Компонентный CLAUDE.md — детали

Каждый компонент получает собственные сфокусированные инструкции:

```markdown
# frontend/CLAUDE.md

## Стек
- Next.js 15 App Router
- Tailwind CSS + shadcn/ui
- React Query для серверного состояния

## Паттерны
- Server Components по умолчанию
- Client Components только для интерактивности
- Используйте app/api/ для API-маршрутов
```

### CLAUDE.local.md — личные предпочтения

Личные переопределения, которые не должны попадать в коммиты:

```markdown
# CLAUDE.local.md (в .gitignore)

- Используй подробные комментарии — я изучаю эту кодовую базу
- Запускай prettier после каждого редактирования файла
- Всегда объясняй логику перед внесением изменений
```

## Система автопамяти

Автопамять — это система Claude Code для запоминания информации между диалогами. Она хранит наблюдения и выводы, которые сохраняются между сессиями.

### Типы

| Тип | Хранится в | Описание |
|-----|-----------|----------|
| `user` | `~/.claude/memory/` | Предпочтения пользователя и глобальные паттерны — применяется ко всем проектам |
| `feedback` | `~/.claude/memory/` | Корректировки и поведенческие правки из обратной связи пользователя |
| `project` | `~/.claude/projects/<hash>/memory/` | Контекст и выводы, специфичные для проекта |
| `reference` | `~/.claude/projects/<hash>/memory/` | Справочные материалы, которые Claude счёл полезными для этого проекта |

### Как это работает

1. Claude наблюдает паттерн, предпочтение или корректировку во время сессии
2. Если наблюдение вероятно пригодится в будущих сессиях, Claude сохраняет его как автопамять
3. При старте следующей сессии релевантные записи автопамяти загружаются в контекст
4. Пользователи могут просматривать и управлять записями через `/memory`

### Когда использовать автопамять, а когда CLAUDE.md

| Используйте автопамять, когда | Используйте CLAUDE.md, когда |
|-------------------------------|------------------------------|
| Инструкция возникла из обратной связи во время сессии | Инструкция запланирована заранее |
| Это личное предпочтение (тон, многословность, рабочий процесс) | Это командное соглашение (стандарты кода, архитектура) |
| Оно эволюционирует со временем через использование | Оно стабильно и проверяется на код-ревью |
| Применяется к пользователю во всех проектах | Применяется к проекту независимо от пользователя |

## Память Agent

Agent с полем `memory:` во frontmatter поддерживают постоянную память, которая сохраняется между сессиями. Это позволяет Agent обучаться и совершенствоваться со временем.

### Области

| Область | Хранится в | Общая | Описание |
|---------|-----------|-------|----------|
| `user` | `~/.claude/agent-memory/<agent>/` | Нет | Кроссроектная память для этого Agent — следует за пользователем |
| `project` | `.claude/agent-memory/<agent>/` | Да | Память, специфичная для проекта — делится с командой через git |
| `local` | `.claude/agent-memory/<agent>/` (в .gitignore) | Нет | Специфичная для проекта, личная память |

### Конфигурация

```yaml
# .claude/agents/code-reviewer.md
---
name: code-reviewer
description: "PROACTIVELY review code changes"
memory: project
---
Review code changes. After each review, note patterns you see frequently
so you can flag them earlier in future reviews.
```

### Примеры использования

- **Код-ревьюер** с `memory: project` — изучает командные паттерны, частые баги, стилевые предпочтения
- **Исследовательский ассистент** с `memory: user` — запоминает исследованные темы, предпочтительные источники, стиль написания
- **Agent для деплоя** с `memory: project` — отслеживает историю деплоев, известные особенности окружения

## Правила (.claude/rules/)

Правила — это markdown-файлы с привязкой к glob-паттернам, которые загружаются автоматически, когда Claude обращается к файлам, совпадающим с паттерном.

### Расположение

| Путь | Область действия |
|------|-----------------|
| `.claude/rules/*.md` | Правила уровня проекта (в git, командные) |
| `~/.claude/rules/*.md` | Глобальные правила (личные, применяются ко всем проектам) |

### Формат

Каждый файл правила начинается со строки `# Glob:`, определяющей, когда правило активируется:

```markdown
# Glob: **/*.tsx

## Правила React-компонентов

- Всегда используйте функциональные компоненты
- Экспортируйте компоненты как именованные экспорты, а не default
- Размещайте стили рядом в файле .module.css
```

```markdown
# Glob: **/*.test.*

## Правила тестирования

- Используйте блоки describe/it, а не test()
- Мокайте внешние зависимости, а не внутренние модули
- Проверяйте поведение, а не реализацию
```

### Когда использовать правила, а когда CLAUDE.md

| Используйте правила, когда | Используйте CLAUDE.md, когда |
|---------------------------|------------------------------|
| Инструкции применяются к определённым типам файлов | Инструкции применяются ко всему проекту |
| Вы хотите ленивую загрузку при обращении к файлам | Вы хотите нетерпеливую загрузку при старте сессии |
| Руководства привязаны к паттерну (`*.py`, `*.tsx`, `migrations/*.sql`) | Руководства универсальны (соглашения по git, архитектурные решения) |

## Лучшие практики

### Ограничьте CLAUDE.md до 200 строк

Длинные файлы CLAUDE.md вызывают раздувание контекста, и Claude может игнорировать инструкции ближе к концу. Разбивайте большие файлы на компонентные CLAUDE.md и правила.

### Используйте правила для руководств по типам файлов

Не пишите «при редактировании файлов Python делайте X» в CLAUDE.md. Вместо этого создайте `.claude/rules/python.md` с `# Glob: **/*.py`. Это загружается только при необходимости.

### Используйте память Agent для развивающихся Agent

Если Agent должен становиться умнее со временем, задайте ему область `memory:`. Agent сохраняет наблюдения, улучшающие будущие запуски — паттерны ревью, запахи кода, особенности деплоя.

### Сжимайте контекст при 50% использования

Периодически запускайте `/context` для проверки использования. Когда оно достигает 50%, выполните `/compact` с необязательными инструкциями фокуса: `/compact focus on the authentication refactor`. Это сохраняет важный контекст, освобождая пространство.

### Добавляйте личные файлы в .gitignore

Всегда добавляйте в `.gitignore`:

```
CLAUDE.local.md
.claude/settings.local.json
```

### Разделяйте стабильное и эволюционирующее

- **Стабильное** (стандарты кода, архитектура) → CLAUDE.md и правила
- **Эволюционирующее** (предпочтения, паттерны, выводы) → автопамять и память Agent

## Устаревание и очистка

### Признаки устаревшей памяти

- CLAUDE.md ссылается на устаревшие API или удалённые функции
- Записи автопамяти противоречат текущим инструкциям в CLAUDE.md
- Память Agent содержит устаревшие паттерны, специфичные для проекта
- Правила ссылаются на паттерны файлов, которых больше нет в кодовой базе

### Стратегии очистки

1. **Аудит CLAUDE.md ежеквартально** — прочитайте каждую инструкцию и проверьте, актуальна ли она
2. **Ревью автопамяти** — запустите `/memory` и удалите записи, которые больше не актуальны
3. **Обрезка памяти Agent** — проверьте `.claude/agent-memory/` на устаревшие наблюдения
4. **Обновление правил** — убедитесь, что glob-паттерны всё ещё совпадают с файлами проекта
5. **Отслеживание через комментарии** — добавляйте `<!-- Last reviewed: 2026-04-09 -->` к разделам CLAUDE.md
