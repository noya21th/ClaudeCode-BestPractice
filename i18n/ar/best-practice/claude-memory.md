## 2. CLAUDE.md في المستودعات الأحادية الكبيرة

عند العمل مع Claude Code في مستودع أحادي، فإن فهم كيفية تحميل ملفات CLAUDE.md في السياق أمر بالغ الأهمية لتنظيم تعليمات مشروعك بفعالية.

<p align="center">
  <a href="https://x.com/bcherny/status/2016339448863355206"><img src="../../../best-practice/assets/claude-memory/claude-memory-monorepo.jpg" alt="تحميل CLAUDE.md في المستودعات الأحادية" width="600"></a>
</p>

### آليتا التحميل

يستخدم Claude Code آليتين مختلفتين لتحميل ملفات CLAUDE.md:

#### التحميل الصاعد (أعلى شجرة المجلدات)

عند بدء Claude Code، يتنقل **صعوداً** من مجلد العمل الحالي نحو جذر نظام الملفات ويحمّل كل CLAUDE.md يجده في الطريق. هذه الملفات تُحمّل **فوراً عند بدء التشغيل**.

#### التحميل الفرعي (أسفل شجرة المجلدات)

ملفات CLAUDE.md في المجلدات الفرعية أسفل مجلد العمل الحالي **لا تُحمّل عند الإطلاق**. تُضمّن فقط عندما يقرأ Claude ملفات في تلك المجلدات الفرعية أثناء جلستك. يُعرف هذا بـ**التحميل الكسول**.

### مثال على هيكل مستودع أحادي

تأمل مستودعاً أحادياً نموذجياً بمجلدات منفصلة لمكونات مختلفة:

```
/mymonorepo/
├── CLAUDE.md          # تعليمات مستوى الجذر (مشتركة عبر جميع المكونات)
├── frontend/
│   └── CLAUDE.md      # تعليمات خاصة بالواجهة الأمامية
├── backend/
│   └── CLAUDE.md      # تعليمات خاصة بالخلفية
└── api/
    └── CLAUDE.md      # تعليمات خاصة بـ API
```

### السيناريو 1: تشغيل Claude Code من مجلد الجذر

عند تشغيل Claude Code من `/mymonorepo/`:

```bash
cd /mymonorepo
claude
```

| الملف | يُحمّل عند الإطلاق؟ | السبب |
|-------|---------------------|-------|
| `/mymonorepo/CLAUDE.md` | نعم | إنه مجلد العمل الحالي |
| `/mymonorepo/frontend/CLAUDE.md` | لا | يُحمّل فقط عند قراءة/تحرير ملفات في `frontend/` |
| `/mymonorepo/backend/CLAUDE.md` | لا | يُحمّل فقط عند قراءة/تحرير ملفات في `backend/` |
| `/mymonorepo/api/CLAUDE.md` | لا | يُحمّل فقط عند قراءة/تحرير ملفات في `api/` |

### السيناريو 2: تشغيل Claude Code من مجلد مكوّن

عند تشغيل Claude Code من `/mymonorepo/frontend/`:

```bash
cd /mymonorepo/frontend
claude
```

| الملف | يُحمّل عند الإطلاق؟ | السبب |
|-------|---------------------|-------|
| `/mymonorepo/CLAUDE.md` | نعم | إنه مجلد أسلاف |
| `/mymonorepo/frontend/CLAUDE.md` | نعم | إنه مجلد العمل الحالي |
| `/mymonorepo/backend/CLAUDE.md` | لا | فرع مختلف من شجرة المجلدات |
| `/mymonorepo/api/CLAUDE.md` | لا | فرع مختلف من شجرة المجلدات |

### النقاط الرئيسية

1. **الأسلاف تُحمّل دائماً عند بدء التشغيل** — Claude يتنقل صعوداً في شجرة المجلدات ويحمّل جميع ملفات CLAUDE.md التي يجدها. هذا يضمن الوصول دائماً لتعليمات مستوى الجذر على مستوى المستودع.

2. **الفروع تُحمّل بكسل** — ملفات CLAUDE.md في المجلدات الفرعية تُحمّل فقط عند التفاعل مع ملفات في تلك المجلدات. هذا يمنع السياق غير ذي الصلة من تضخيم جلستك.

3. **الأشقاء لا يُحمّلون أبداً** — إذا كنت تعمل في `frontend/`، لن تحصل على `backend/CLAUDE.md` أو `api/CLAUDE.md` محمّلة في السياق.

4. **CLAUDE.md العام** — يمكنك أيضاً وضع CLAUDE.md في `~/.claude/CLAUDE.md` في مجلد المنزل، والذي ينطبق على جميع جلسات Claude Code بغض النظر عن المشروع.

### لماذا يعمل هذا التصميم للمستودعات الأحادية

- **التعليمات المشتركة تنتشر للأسفل** — CLAUDE.md في مستوى الجذر يحتوي على اتفاقيات على مستوى المستودع ومعايير الترميز والأنماط الشائعة التي تنطبق في كل مكان.

- **التعليمات الخاصة بالمكونات تبقى معزولة** — مطورو الواجهة الأمامية لا يحتاجون لتعليمات الخلفية تشوّش سياقهم، والعكس صحيح.

- **السياق محسّن** — بتحميل ملفات CLAUDE.md الفرعية بكسل، يتجنب Claude Code تحميل مئات الكيلوبايتات من التعليمات غير ذات الصلة عند بدء التشغيل.

### أفضل الممارسات

1. **ضع الاتفاقيات المشتركة في CLAUDE.md الجذر** — معايير الترميز وتنسيقات رسائل commit وقوالب PR والإرشادات الأخرى على مستوى المستودع.

2. **ضع التعليمات الخاصة بالمكونات في CLAUDE.md المكوّن** — أنماط الإطار الخاصة وبنية المكونات واتفاقيات الاختبار الفريدة لذلك المكوّن.

3. **استخدم CLAUDE.local.md للتفضيلات الشخصية** — أضفه إلى `.gitignore` للتعليمات التي لا ينبغي مشاركتها مع الفريق.
