## الأنماط المضادة في البنية

### 1. CLAUDE.md أحادي ضخم

**المشكلة**: ملف CLAUDE.md واحد يتجاوز 500 سطر يحتوي على تعليمات التثبيت، واتفاقيات الترميز، ووثائق API، وخطوات النشر، ونصائح تصحيح الأخطاء مختلطة معاً.

**الأعراض**: يتجاهل Claude التعليمات القريبة من نهاية الملف. التزام غير متسق بالاتفاقيات. تضخّم السياق عند بدء الجلسة.

**الحل**: قسّم إلى ملفات مركّزة. أبقِ CLAUDE.md الجذري تحت 200 سطر للاتفاقيات المشتركة. استخدم ملفات CLAUDE.md على مستوى المكوّنات للتفاصيل. استخدم `.claude/rules/*.md` مع أنماط glob للإرشادات الخاصة بأنواع الملفات.

```
# Before (monolithic)
CLAUDE.md              # 800 lines covering everything

# After (split)
CLAUDE.md              # 150 lines — shared conventions, project overview
frontend/CLAUDE.md     # 80 lines — React patterns, component structure
backend/CLAUDE.md      # 80 lines — API conventions, database patterns
.claude/rules/ts.md    # TypeScript-specific rules (Glob: **/*.ts)
.claude/rules/test.md  # Testing conventions (Glob: **/*.test.*)
```

### 2. Agent لكل شيء

**المشكلة**: استخدام Agents لمهام بسيطة تحتاج فقط حقن تعليمات في السياق الحالي.

**الأعراض**: عزل سياق غير ضروري. النتائج غير مرئية في المحادثة الرئيسية. تنفيذ بطيء بسبب عبء بدء تشغيل الـ Agent. هدر في الرموز بسبب تكرار السياق.

**الحل**: استخدم Skill للمعرفة المتخصصة والإجراءات القابلة لإعادة الاستخدام. استخدم Command لقوالب الموجّهات المُفعّلة من المستخدم. احتفظ بـ Agents للمهام التي تحتاج فعلاً عزلاً أو ذاكرة أو تقييد أدوات أو تشغيلاً في الخلفية.

### 3. غياب عزل السياق

**المشكلة**: تشغيل مهام استكشافية طويلة (بحث، مراجعة كود، تحليل بيانات) مباشرة في المحادثة الرئيسية دون عزل.

**الأعراض**: يمتلئ سياق المحادثة بسرعة. نتائج وسيطة غير ذات صلة تلوث الردود المستقبلية. الحاجة إلى `/compact` بشكل متكرر.

**الحل**: استخدم Agent للمهام التي تولّد مخرجات وسيطة كبيرة. يعمل الـ Agent في سياق جديد ويعيد الملخص فقط.

### 4. استدعاء دائري بين الـ Agents

**المشكلة**: Agent A يستدعي Agent B، الذي يستدعي Agent A، مما يخلق حلقة لا نهائية.

**الأعراض**: الوصول إلى حد `maxTurns`. استنفاد ميزانية الرموز. لا مخرجات مفيدة.

**الحل**: صمّم سير عمل DAG (رسم بياني موجّه غير دوري). استخدم حقل `tools:` لتقييد الـ Agents التي يمكن للـ Agent الفرعي استدعاءها. راجع تعريفات الـ Agents بحثاً عن مراجع `Agent(...)` دائرية.

```yaml
# Restrict spawnable agents
---
name: planner
tools: Read, Glob, Grep, Agent(implementer)
# planner can only spawn implementer, not itself
---
```

### 5. أسماء نماذج مكتوبة يدوياً

**المشكلة**: استخدام معرّفات نماذج كاملة مثل `claude-sonnet-4-20250514` في تعريفات الـ Agent.

**الأعراض**: تتعطل الـ Agents عند إيقاف معرّفات النماذج. سلوك غير متسق عبر البيئات.

**الحل**: استخدم الأسماء المستعارة: `haiku`، `sonnet`، `opus`، أو `inherit`. هذه تُحلّ دائماً إلى أحدث إصدار.

## الأنماط المضادة في التكوين

### 6. أذونات مفرطة التساهل

**المشكلة**: تعيين `"permissionMode": "bypassPermissions"` على جميع الـ Agents واستخدام `--dangerously-skip-permissions` كإعداد افتراضي.

**الأعراض**: حذف ملفات دون مراجعة. `git push --force` غير مقصود. مخاطر أمنية من أدوات MCP غير موثوقة. لا سجل تدقيق لموافقات الأدوات.

**الحل**: استخدم `acceptEdits` للـ Agents التي تحتاج كتابة ملفات لكن لا ينبغي أن تشغّل أوامر عشوائية. استخدم `plan` للـ Agents المخصصة للبحث فقط. احتفظ بـ `bypassPermissions` فقط لبيئات CI/CD مع الحماية.

| الوضع | التعديلات | Bash | الأفضل لـ |
|-------|----------|------|----------|
| `default` | يسأل | يسأل | التطوير التفاعلي |
| `acceptEdits` | تلقائي | يسأل | Agents كتابة الملفات |
| `plan` | يرفض | يرفض | Agents البحث والتخطيط |
| `bypassPermissions` | تلقائي | تلقائي | CI/CD مع حماية فقط |

### 7. تجاهل تسلسل الإعدادات

**المشكلة**: تعريف الإعدادات في الملف الخاطئ والتساؤل لماذا يتم تجاوزها أو تجاهلها.

**الأعراض**: Hooks لا تعمل. الأذونات ليست كما تتوقع. أعضاء الفريق لديهم سلوك مختلف.

**الحل**: افهم ترتيب الأولوية (الأعلى يفوز):

1. **المُدارة** (`managed-settings.json`) — مفروضة من المؤسسة
2. **أعلام CLI** — تجاوزات لجلسة واحدة
3. **محلية** (`.claude/settings.local.json`) — إعدادات مشروع شخصية
4. **مشتركة** (`.claude/settings.json`) — إعدادات مشروع مشتركة للفريق
5. **عامة** (`~/.claude/settings.json`) — إعدادات افتراضية شخصية

### 8. جميع الـ Hooks متزامنة

**المشكلة**: كل Hook يعمل بشكل متزامن، مما يعطّل رد Claude أثناء تنفيذ السكربتات.

**الأعراض**: ردود بطيئة. توقف ملحوظ بعد كل استدعاء أداة. المستخدم يشعر أن Claude بطيء.

**الحل**: عيّن `"async": true` على الـ Hooks التي تنفذ تأثيرات جانبية (التسجيل، الإشعارات، القياس عن بُعد). أبقِ الـ Hooks متزامنة فقط عندما تحتاج التحكم في السلوك أو تعديله (فحوصات الأمان، قرارات الأذونات).

### 9. لا يوجد settings.local.json

**المشكلة**: التفضيلات الشخصية (إشعارات صوتية، مسارات مخصصة، تسجيل تصحيح الأخطاء) مُرسلة إلى `.claude/settings.json` حيث تؤثر على الفريق بأكمله.

**الأعراض**: زملاء الفريق يسمعون أصوات إشعاراتك. Hooks التصحيح تعمل في الإنتاج. خوادم MCP الشخصية تظهر للجميع.

**الحل**: استخدم `.claude/settings.local.json` للتجاوزات الشخصية. تأكد أنه في `.gitignore`. شارك فقط الـ Hooks والإعدادات التي يحتاجها الفريق بأكمله.

## الأنماط المضادة في الـ Skills

### 10. user-invocable: true على Skills مخصصة للـ Agent فقط

**المشكلة**: Skill مخصص فقط للتحميل المسبق بواسطة Agent (عبر حقل `skills:`) يظهر في قائمة `/` الخاصة بالمستخدم.

**الأعراض**: المستخدمون يستدعون الـ Skill مباشرة ويحصلون على نتائج مربكة لأنه صُمّم كمعرفة خلفية لـ Agent.

**الحل**: عيّن `user-invocable: false` على الـ Skills المخصصة فقط للتحميل المسبق بواسطة الـ Agent.

```yaml
---
name: internal-api-conventions
user-invocable: false
description: "REST API conventions for the internal platform"
---
```

### 11. غياب الوصف

**المشكلة**: Skill بدون حقل `description` في الـ frontmatter الخاص به.

**الأعراض**: لا يستطيع Claude اكتشاف الـ Skill تلقائياً. يعمل فقط عندما يكتب المستخدم `/slash-command` صراحةً. الـ Skill غير مرئي لمنطق الاستدعاء التلقائي.

**الحل**: أضف دائماً `description` يوضّح **متى** يُستخدم الـ Skill. استخدم كلمات مفتاحية يصادفها Claude بشكل طبيعي: "Use when creating React components" أو "Triggers on Python test files."

### 12. ملفات Skill ضخمة

**المشكلة**: ملف SKILL.md واحد بأكثر من 1000 سطر يغطي كل حالة حدّية ومثال ومرجع.

**الأعراض**: تضخّم السياق عند تحميل الـ Skill. قد يتجاهل Claude التعليمات القريبة من النهاية. بطيء التحميل عند التحميل المسبق في الـ Agents.

**الحل**: أبقِ SKILL.md مركّزاً على الإجراء الأساسي (أقل من 200 سطر). انقل المواد المرجعية إلى ملفات `references/` داخل مجلد الـ Skill. استخدم استيراد `@path` للمحتوى التكميلي الذي يحمّله Claude عند الطلب.

### 13. disable-model-invocation على Skills مطلوبة بشكل متكرر

**المشكلة**: تعيين `disable-model-invocation: true` على Skill ينبغي أن يكتشفه Claude ويستدعيه تلقائياً.

**الأعراض**: لا يستدعي Claude الـ Skill تلقائياً أبداً. يجب على المستخدمين دائماً كتابة `/slash-command` يدوياً حتى عندما يستدعي السياق الـ Skill بوضوح.

**الحل**: استخدم `disable-model-invocation: true` فقط على الـ Skills المُفعّلة من المستخدم حصراً (عمليات خطيرة، نشر، إلخ). للـ Skills التي ينبغي أن يكتشفها Claude من تلقاء نفسه، اتركها بدون تعيين أو عيّنها إلى `false`.

## الأنماط المضادة التشغيلية

### 14. عدم الضغط أبداً

**المشكلة**: تشغيل جلسة طويلة دون ضغط، مما يترك السياق يمتلئ إلى 100%.

**الأعراض**: يبدأ Claude بنسيان التعليمات السابقة. تصبح الردود أقل دقة. الجلسة تخطئ في النهاية بتجاوز السياق.

**الحل**: شغّل `/compact` يدوياً عند حوالي 50% من استخدام السياق. استخدم `/context` لتصوّر الاستخدام الحالي. للمهام الطويلة، قسّم العمل إلى مهام فرعية يكتمل كل منها في أقل من 50% من سعة السياق.

### 15. لا حلقات تحقق

**المشكلة**: قبول مخرجات Claude دون التحقق من عملها — لا تشغيل اختبارات، لا فحص أنواع، لا فحص بصري.

**الأعراض**: الكود المولّد يحتوي على أخطاء نحوية. تكامل API يستخدم نقاط نهاية خاطئة. ملفات التكوين تحتوي على أخطاء مطبعية.

**الحل**: تحقق دائماً من المخرجات. شغّل الاختبارات بعد توليد الكود. استخدم أتمتة المتصفح (Chrome، Playwright) للتحقق البصري. أضف خطوات التحقق في أوامرك وتعريفات الـ Agent.

### 16. تجاهل حدود المعدل

**المشكلة**: تخطيط مهام متعددة الـ Agents كبيرة دون مراعاة ميزانية الرموز وحدود المعدل.

**الأعراض**: الـ Agents تصطدم بحدود المعدل في منتصف المهمة. العمل يضيع أو يكون غير مكتمل. مفاجآت في الفوترة.

**الحل**: استخدم `/usage` للتحقق من حالة حد المعدل الحالي. خطط المهام لتتناسب مع ميزانيتك. استخدم `model: haiku` للمهام الفرعية عالية الحجم منخفضة التعقيد. قسّم المهام الكبيرة إلى جلسات أصغر.
